# HRIS Data Validation & Reporting Support

## Overview
This project demonstrates hands-on experience supporting HR systems through data validation, reporting, and documentation. The focus is on ensuring data accuracy, identifying inconsistencies, and producing reliable reporting outputs that support HR operations and decision-making.

The work reflects responsibilities commonly found in HRIS Analyst, Business Systems Analyst, and Application Support roles.

## Dataset
The dataset represents employee-level HR data commonly stored in HRIS platforms, including job roles, departments, tenure, compensation, and employment status.

## Objectives
- Validate HR data for completeness and accuracy
- Identify inconsistencies that could impact reporting or downstream systems
- Support reporting needs through structured SQL queries
- Demonstrate clear documentation and repeatable analysis

## Key Activities
- Data profiling and validation checks using SQL
- Identification of missing, inconsistent, or out-of-range values
- Creation of reporting-ready queries for HR metrics
- Documentation of findings and assumptions

## Tools Used
- SQL
- Excel / Power BI
- GitHub

## Example Use Cases
- HRIS data audits
- Reporting support for HR and leadership
- Pre-validation before payroll or analytics reporting
